3. Server Maintenance Documentation
A significant infrastructure overhaul was conducted between 2025-10-15 and 2025-11-20. This maintenance period involved back-end database optimization and security patching.

Maintenance log summary:
- Phase 1 (Oct 15–Oct 28): Database indexing and archival of 2024 legacy data.
- Phase 2 (Oct 29–Nov 10): Load balancer reconfiguration and hardware firmware updates.
- Phase 3 (Nov 11–Nov 20): Final stress testing and security protocol deployment.

4. Technical Impact Analysis: Website Loading Speed
The IT department tracked website performance metrics during the maintenance window (2025-10-15 to 2025-11-20) to determine user experience impact. 

During Phase 1 and 2, the average website loading speed increased from 1.2 seconds to 3.8 seconds due to server-side resource allocation during indexing. Post-maintenance (after Nov 20), consistent loading speeds of 0.9 seconds were achieved, representing a 25% improvement over pre-maintenance baselines. The sales volume during the maintenance period showed a minor 4% dip, which is attributed to these temporary performance latencies.

6. Proposals for Future System Migration
To sustain long-term growth and handle projected increases in technical requirements, the IT Department proposes a full-scale system migration scheduled for late 2026. 

Proposal A: Cloud-Native Transition
Shifting all internal databases and inventory management to a distributed cloud architecture. This aims to eliminate the need for the intensive manual maintenance periods seen in October 2025.

Proposal B: Hybrid Modular Infrastructure
A phased migration that keeps core lab proprietary data on-site while moving front-facing sales portals to high-availability servers. This proposal prioritizes security and minimized downtime during the transition phase.
